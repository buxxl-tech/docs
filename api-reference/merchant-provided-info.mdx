---
title: '接入准备'
description: '商户在接入 WaaS 系统时，需要提供以下信息：'
---

- **1、商户邮箱：**
  用于登录 WaaS 后台管理系统，并在商户进行操作时接收验证码以完成身份验证。

- **2、回调 URL：**
  回调 URL 是商户系统接收 WaaS 系统通知的关键接口，用于以下场景：
  - 存币通知
  - 提币结果回调
  - 更换用户钱包地址通知

  **说明：**
  WaaS 在特定事件发生时会向商户提供的回调 URL 发送通知，告知操作结果或状态变化。商户需按规则实现回调接口，确保及时接收并处理通知。

  **详情请参阅：**[Webhook 回调](/api-reference/webhook/webhook-callback)


- **3、提现请求地址（可选）：**
  提现请求地址的用途取决于是否启用二次确认机制：
  - **无需二次确认**：商户调用 `api/order/withdrawWithoutVerify` 接口，一次性提交完整提币订单信息给 WaaS。
  - **需要二次确认**：商户调用 `api/order/withdrawRequest` 接口，仅通知 WaaS 有用户发起了提币操作，具体提币信息需通过二次确认提供。

  二次确认机制可根据商户需求启用，通过分离提币请求与信息确认，增强通信安全性，防止数据篡改和非法操作。

  **详情请参阅：**[提现二次确认回调](/api-reference/webhook/withdraw-confirmation-callback)

- **4、IP 白名单：**
 商户需提供服务器的固定 IP 地址，用于添加到 IP 白名单中。只有在 IP 白名单中的地址才能够访问 API 接口，以确保系统的安全性。
