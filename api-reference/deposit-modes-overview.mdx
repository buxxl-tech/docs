### 1. 共享地址模式

**描述**：
在共享地址模式下，商户的所有客户使用同一个指定的钱包地址进行存款，存款金额可以指定或不指定。

**适用场景**：
- 收款频率较低的商户。
- 对资金安全性要求不高的商户。

**特点**：
- 客户使用**统一的存款地址**进行充值。
- 需要通过**充值金额的唯一性**（如小数点金额差异）来区分不同客户/订单。
- 实现成本较低，无需为每个客户创建独立地址。

---

### 2. 独立地址模式

**描述**：
在独立地址模式下，商户为其每个客户分配独立的钱包地址进行存款，无需指定存款金额的唯一性。

**适用场景**：
- 收款频率较高的商户。
- 对资金安全性要求较高的商户。

**特点**：
- 每个客户拥有**独立的钱包地址**，无需通过金额区分客户/订单。
- 增强了资金安全性，支持更灵活的钱包管理。
- 需要对资金进行归集操作，可能增加归集成本。
